note

This page covers how to convert existing code from Java Spec API to Kotlin API. Please refer to this page for the main Layout System docs.

Rows and Columns, Revisitedâ€‹

As in the Spec API, Row and Column (see the image, below) are the primary layout containers used in the KComponent API. If you need a refresher on them, take a look a the Layout System with Flexbox page.

Differences between the KComponent API and the Spec APIâ€‹

If you're used to the Spec way of writing containers, there are a few important differences when using Row and Column in the KComponent API:

  1. Props of Rows and Columns - props that configure the Row/Column itself, like alignItems or justifyContent, now appear as props directly on the container Component. In this way, there is no difference from setting props on any other component.
  2. Common Props - props that configure a particular child's positioning within the context of a Row/Column, like alignSelf or flex, now appear with other common props on the Style passed to that child. You can think of this as akin to LayoutParams in vanilla Android - they must be able to go on any component and are read by the parent.
  3. Children - children are added within a trailing lambda using the child() call.

You can see this in practice in the following code:

 Column(style = Style.padding(all = 8.dp), alignItems = YogaAlign.CENTER) {
   child(Text(style = Style.flex(grow = 1f), text = "Foo"))
   child(Text(text = "Bar"))
 }

Migrating from the Spec APIâ€‹

Why are Layout Props Now Defined in Two Different Ways?

Without code-generated builders, Bloks had to make a distinction between props of the container itself and common props that can be set on children of the container.

For example, alignItems configures the default alignment for children in this container and is only valid on a flexbox container. It's set like a normal component prop on Row/Column.

On the other hand, alignSelf can be respected on any child of a container, whether it's a Text, a Switch, or some custom Component: it's therefore exposed as a common prop via Style.

The benefit of this is that you can now statically verify all required props are set instead of verifying at runtime (as well as not rely on Codegen!).

A property is set via Style if and only if it configures a specific child!

Example Migrationâ€‹

Below is an example of a simple Component using various layout properties, converted to an equivalent KComponent:

  • Kotlin
  • Java
class FlexboxComponent(
    private val username: String,
    @DrawableRes private val avatarRes: Int,
    @DrawableRes private val imageRes: Int
) : KComponent() {
  override fun ComponentScope.render(): Component? {
    return Column {
      child(
          Row(alignItems = YogaAlign.CENTER, style = Style.padding(all = 8.dp)) {
            child(
                Image(
                    drawable = drawableRes(avatarRes),
                    style = Style.width(36.dp).height(36.dp).margin(start = 4.dp, end = 8.dp)))
            child(Text(text = username, textStyle = Typeface.BOLD))
          })
      child(
          Image(
              drawable = drawableRes(imageRes),
              scaleType = ImageView.ScaleType.CENTER_CROP,
              style = Style.aspectRatio(1f)))
    }
  }
}
@LayoutSpec
public class JavaApiComponentSpec {

  @OnCreateLayout
  static Component onCreateLayout(
      ComponentContext c, @Prop String username, @Prop int avatarRes, @Prop int imageRes) {
    return Column.create(c)
        .child(
            Row.create(c)
                .paddingDip(YogaEdge.ALL, 8)
                .alignItems(YogaAlign.CENTER)
                .child(
                    Image.create(c)
                        .drawableRes(avatarRes)
                        .widthDip(36)
                        .heightDip(36)
                        .marginDip(YogaEdge.START, 4)
                        .marginDip(YogaEdge.END, 8))
                .child(Text.create(c).text(username).textStyle(Typeface.BOLD)))
        .child(
            Image.create(c)
                .drawableRes(imageRes)
                .scaleType(ImageView.ScaleType.CENTER_CROP)
                .aspectRatio(1))
        .build();
  }
}
